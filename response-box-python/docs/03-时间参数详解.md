# 反应盒时间参数详细说明

## 📊 参数概览

反应盒可以输出以下4个关键时间参数：

| 参数 | 含义 | 说明 |
|------|------|------|
| **t1** | 按键按下反应时 | 从刺激呈现到按键**按下**的时间（最重要！） |
| **t2** | 按键松开时间 | 从刺激呈现到按键**松开**的时间 |
| **软件系统反应时** | Python测量的反应时 | 从发送触发命令到接收完数据的时间 |
| **系统差值** | 通信和处理延迟 | 软件反应时 - t2（串口通信+Python处理） |

## 🔍 参数详解

### 1. t1 - 按键按下反应时 ⭐最重要⭐

```
触发信号 (0xFB) ────┬──→ 按键按下 ←── t1
                    │
                    └──→ 按键松开 ←── t2
```

- **定义**：从触发0xFB命令到被试按下按键的时间
- **典型值**：200-600 ms（取决于任务难度）
- **用途**：这是实验中最主要的反应时测量值
- **特点**：由反应盒硬件直接测量，精度高（1ms分辨率）

### 2. t2 - 按键松开时间

- **定义**：从触发0xFB命令到被试松开按键的时间
- **关系**：t2 > t1（必然大于t1）
- **计算**：t2 - t1 = 按键按压持续时间
- **典型值**：比t1大100-300ms（取决于被试按键习惯）

### 3. 软件系统反应时

```python
trigger_time = time.perf_counter()  # 发送触发命令
ser.write(b'\xFB\x00')
# ... 等待被试按键 ...
frame = ser.read(8)                 # 接收完整数据
receive_time = time.perf_counter()
software_rt = (receive_time - trigger_time) * 1000
```

- **定义**：Python程序从发送触发到接收完数据的总时间
- **包含**：t2 + 串口通信延迟 + Python处理时间
- **典型值**：t2 + 10-30ms

### 4. 系统差值

- **计算**：system_delay = software_rt - t2
- **含义**：串口通信和Python处理造成的额外延迟
- **典型值**：10-30 ms
- **重要性**：校准实验系统的参考值

## 📈 实际示例

假设被试看到刺激后，350ms按下按键，550ms松开按键：

```
时间轴：
0ms    ─── 触发 (0xFB)
         |
350ms   ─── 按键按下  ← t1 = 350ms
         |
550ms   ─── 按键松开  ← t2 = 550ms
         |
575ms   ─── Python接收完数据 ← software_rt = 575ms

计算结果：
  t1 = 350ms                    （真实反应时）
  t2 = 550ms                    （松开时间）
  按压持续时间 = 200ms          （550 - 350）
  软件反应时 = 575ms            （Python测量）
  系统差值 = 25ms               （575 - 550，通信延迟）
```

## ✅ 实验中应该使用哪个参数？

### **强烈推荐：使用 t1**

```python
key, t1, t2, ide = get_reaction_time()

# 保存数据时
trial_data = {
    'reaction_time': t1,      # ✓ 使用t1作为反应时
    'key_pressed': key,
    # ... 其他数据
}
```

**原因：**
1. **准确性**：t1是硬件直接测量的按键按下时刻，精度1ms
2. **标准化**：心理学实验通常定义反应时为"按下"时刻，而非"松开"
3. **可比性**：t1不受被试按键习惯（按压时长）的影响
4. **无延迟**：不包含软件处理延迟，是纯粹的反应时

### 可选：使用 t2

某些特殊情况下可以使用t2：
- 研究按键动作的完整时间过程
- 分析按压持续时间（t2-t1）

### 不推荐：使用 software_rt

不建议直接使用Python测量的软件反应时，因为：
- 包含不可控的系统延迟
- 受串口通信速度影响
- 精度低于硬件测量

## 💡 常见问题

### Q: 为什么我的反应时正常，但系统延迟测试显示很高？

A: 之前的系统延迟测试有bug，已修复。真实系统差值应该在10-30ms范围内。

### Q: t1和t2哪个更准确？

A: 两者都由硬件测量，精度相同（1ms）。但**t1更符合心理学实验对反应时的定义**。

### Q: 系统差值会影响实验结果吗？

A: 不会。因为：
1. 使用t1/t2（硬件测量），不使用软件反应时
2. 系统差值是常数，对所有试次影响相同
3. 实验比较的是条件间差异，常数延迟会被抵消

### Q: 为什么按压持续时间会变化？

A: 这是正常的。按压时长受以下因素影响：
- 被试个体差异
- 任务难度（难题可能按得更重、更久）
- 疲劳程度
- 按键习惯

## 📝 数据记录建议

在实验中完整记录所有参数：

```python
# 获取详细数据
result = get_reaction_time_detailed()

# 保存完整信息
trial_data = {
    'trial_id': trial_num,
    'stimulus': stimulus_info,
    'key': result['key'],
    't1': result['t1'],                      # 主要分析用
    't2': result['t2'],                      # 备用
    'press_duration': result['press_duration'],  # 可选分析
    'software_rt': result['software_rt'],     # 质量检查
    'system_delay': result['system_delay'],   # 质量检查
    # ... 其他实验变量
}
```

**分析时：**
- 主要分析：使用 `t1`
- 数据质量检查：检查 `system_delay` 是否异常
- 探索性分析：可以分析 `press_duration` 的模式
