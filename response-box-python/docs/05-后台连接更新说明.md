# 更新说明 - 后台连接反应盒

## 问题描述

之前的实现存在以下问题：
1. 先连接反应盒，再打开测试窗口，导致连接期间无法进行任何交互
2. 如果连接失败或耗时较长，用户无法通过界面取消操作
3. 连接阻塞主线程，用户体验不佳

## 解决方案

### 修改后的流程

1. **先打开pretest窗口**
2. **在窗口中后台连接反应盒**（使用独立线程）
3. **连接期间**：
   - "退出测试"按钮始终可用（可随时退出）
   - "开始测试"按钮变灰且不可点击
   - 显示"正在连接反应盒..."状态
4. **连接成功后**：
   - "开始测试"按钮变蓝且可点击
   - 显示端口号和设备ID
   - 状态变为"反应盒已连接，可开始测试"（绿色）
5. **连接失败时**：
   - 显示错误信息（红色）
   - "开始测试"按钮保持禁用
   - 可点击"退出测试"离开

### 代码修改

#### 1. `src/pretest.py`

**函数签名修改**：
```python
def run_pretest_window(port=None, device_id="0001", connect_in_background=False):
    """
    参数：
        port: 反应盒端口（如果为None且connect_in_background=True，则在后台连接）
        device_id: 设备ID
        connect_in_background: 是否在窗口中后台连接反应盒
    返回：
        ('proceed', port) 进入实验，返回端口
        ('exit', None)    退出程序
    """
```

**主要改动**：
- 添加后台连接线程
- 动态更新UI状态（端口号、设备ID、连接状态）
- 按钮状态根据连接状态动态变化
- 使用线程锁保护共享状态

#### 2. `experiment_psychopy.py`

**主程序流程修改**：
```python
# 之前：先连接再打开窗口
PORT = auto_find_reaction_box()
decision = run_pretest_window(PORT, device_id=device_id)

# 现在：先打开窗口，在窗口中后台连接
decision, PORT = run_pretest_window(port=None, device_id=None, connect_in_background=True)
```

### 用户体验改进

1. **立即响应**：窗口立即打开，不需要等待连接
2. **可控性**：连接期间可随时退出
3. **可见性**：实时显示连接状态
4. **安全性**：防止在未连接时进入实验
5. **友好性**：连接失败时显示清晰的错误信息

## 使用方式

### 正常使用（后台连接）

```bash
python experiment_psychopy.py
```

1. 填写被试信息
2. 窗口打开，显示"正在连接反应盒..."
3. 等待连接成功（通常2-5秒）
4. 点击"开始测试"测试按键
5. 点击"进入实验"开始正式实验

### 已知端口（跳过后台连接）

如果已知端口，可以直接在代码中指定：

```python
decision, PORT = run_pretest_window(port='COM5', device_id='0001', connect_in_background=False)
```

## 技术细节

### 线程安全

使用 `threading.Lock()` 保护共享状态：

```python
connection_lock = threading.Lock()

with connection_lock:
    is_connected = connection_result['connected']
    current_port = connection_result.get('port')
```

### UI更新

主循环中检查连接状态并更新UI：

```python
if connect_in_background and is_connected and state == "connect":
    # 连接成功，更新UI
    port_text.text = f"当前反应盒端口号：{current_port}"
    status_stim.color = "green"
    btn_start['enabled'] = True
    connect_in_background = False  # 只更新一次
```

### 按钮状态管理

按钮状态通过字典管理，`draw_button`函数根据`enabled`属性动态渲染：

```python
def draw_button(btn):
    btn["rect"].fillColor = "#4a6fa5" if btn["enabled"] else "#9aa8b5"
    btn["text"].opacity = 1.0 if btn["enabled"] else 0.5
```

## 兼容性

- 向后兼容：仍支持直接传入端口号（跳过后台连接）
- 新功能：通过`connect_in_background=True`启用后台连接
- 返回值：现在返回元组`(decision, port)`而非单一字符串

## 注意事项

1. 后台连接线程设置为守护线程（`daemon=True`），窗口关闭时自动退出
2. 连接失败时不会崩溃，而是显示错误信息
3. "进入实验"按钮只在测试过后才启用（需要先点"开始测试"）
