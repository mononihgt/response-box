## 1. 硬件连接与驱动安装

### 1.1 硬件连接

#### 连接方式
- **有线连接（USB串口）**：使用 USB 转串口线连接反应盒和电脑
- **无线连接（2.4G）**：部分型号支持无线接收器（需要确认你的反应盒型号）

#### 连接步骤
1. **有线连接**：
   - 将 USB 串口线插入电脑的 USB 接口
   - 将串口线另一端连接到反应盒
   - 打开反应盒电源开关（如果有）

2. **2.4G 无线连接**（如果支持）：
   - 将 2.4G 无线接收器插入电脑 USB 接口
   - 打开反应盒电源
   - 等待配对指示灯稳定

### 1.2 驱动安装

#### 确认芯片型号
本项目使用的是 **CH341** 芯片的 USB 转串口设备。

#### 安装驱动
1. 找到项目中的 `CH341SER_DRIVER/` 文件夹
2. 根据系统选择对应版本：
   - Windows 7/8/10/11：运行安装程序
   - 如果是自动安装：插入设备后系统会自动识别
3. 安装完成后，在**设备管理器**中查看：
   - 打开设备管理器（Win+X → 设备管理器）
   - 找到 **端口（COM 和 LPT）**
   - 应该能看到类似 `USB-SERIAL CH340 (COM3)` 的设备
   - 记下端口号（如 COM3、COM4 等）

### 1.3 验证连接

#### 方法1：使用设备管理器
```
1. 打开设备管理器
2. 查看"端口(COM和LPT)"
3. 插拔USB线，观察端口是否出现/消失
4. 记录COM端口号（如COM3）
```

## 2. 串口通信基础

### 2.1 核心参数

| 参数 | 值 | 说明 |
|------|-----|------|
| **波特率 (BAUD)** | 115200 | 数据传输速率，必须与反应盒一致 |
| **端口 (PORT)** | COM3/COM4/... | 根据实际情况修改 |
| **超时 (TIMEOUT)** | 60秒 | 等待按键的最长时间 |
| **数据位** | 8 | 默认值 |
| **停止位** | 1 | 默认值 |
| **校验位** | None | 无校验 |

### 2.2 Python串口库

#### 安装pyserial
```bash
pip install pyserial
```

#### 基本使用
```python
import serial

# 打开串口
ser = serial.Serial(
    port='COM3',        # 端口号
    baudrate=115200,    # 波特率
    timeout=1           # 超时时间（秒）
)

# 发送数据
ser.write(b'\x5A\x00')  # 发送字节数据

# 读取数据
data = ser.read(8)      # 读取8字节

# 关闭串口
ser.close()
```

#### 使用上下文管理器（推荐）
```python
import serial

with serial.Serial('COM3', 115200, timeout=1) as ser:
    ser.write(b'\x5A\x00')
    data = ser.read(8)
    # 自动关闭串口
```

## 3. 反应盒通信协议

### 3.1 协议概述

反应盒使用**串口通信协议**，采用**字节流**方式传输数据。

### 3.2 命令协议

#### 查询设备编号
```python
# 发送命令
命令: 0x5A 0x00

# 返回格式
[0x5A][设备ID高字节][设备ID低字节]
例如: 5A 12 34  表示设备ID为0x1234
```

#### 触发计时
```python
# 停止之前的计时
ser.write(b'\xFC\x00')

# 开始计时
ser.write(b'\xFB\x00')
```

#### 读取按键响应
```python
# 读取8字节数据帧
frame = ser.read(8)

# 数据帧格式（8字节）
[FB] [按键号] [t1高] [t1低] [t2高] [t2低] [00] [01]
 |      |        |              |        |
 |      |        |              |        └─ 校验字节
 |      |        |              └─ 第二段时间(t2)
 |      |        └─ 第一段时间(t1)
 |      └─ 按键编号(0x01-0x05)
 └─ 帧头(固定0xFB)
```

### 3.3 数据解析

#### 字节序
- 采用**大端字节序（Big Endian）**
- 高字节在前，低字节在后

#### 解析示例
```python
import struct

# 假设收到的帧数据
frame = b'\xFB\x01\x00\x64\x00\xC8\x00\x01'

# 解析帧头和按键号
if frame[0] == 0xFB:
    key_num = frame[1]  # 按键编号
    
    # 解析时间（大端16位无符号整数）
    t1, t2 = struct.unpack('>HH', frame[2:6])
    
    print(f"按键号: {key_num}")
    print(f"时间t1: {t1} ms")
    print(f"时间t2: {t2} ms")
```

## 4. 按键对应关系

### 4.1 按键编号

| 按键编号 | 颜色 | 用途 | 说明 |
|---------|------|------|------|
| 0x01 | 绿色 | 响应1 | 通常表示"是"、"正确"、"积极" |
| 0x02 | 蓝色 | 响应2 | 备用按键 |
| 0x03 | 红色 | 响应3 | 通常表示"否"、"错误"、"消极" |
| 0x04 | 黄色 | 响应4 | 备用按键 |
| 0x05 | 白色 | 退出 | 紧急退出实验 |

## 5. 反应时参数提取

### 5.1 时间参数说明

反应盒返回两个时间参数：**t1** 和 **t2**

| 参数 | 含义 | 单位 | 说明 |
|------|------|------|------|
| **t1** | 刺激呈现到按键**按下**的时间 | 毫秒 (ms) | 主要的反应时测量值，最重要！ |
| **t2** | 刺激呈现到按键**松开**的时间 | 毫秒 (ms) | 包含按键持续时间 |
| **t2-t1** | 按键按压持续时间 | 毫秒 (ms) | 按键从按下到松开的时间 |

**重要理解：**
- `t1` 是按键**按下**的时刻（真正的反应时）
- `t2` 是按键**松开**的时刻（t2 > t1）
- 软件测量的反应时 ≈ t2 + 系统延迟（串口通信和Python处理时间）
- 在实验中应该使用 **t1** 作为被试的真实反应时

### 5.2 反应时计算

```python
def get_reaction_time():
    """
    获取反应时数据
    
    返回:
        res: 按键编号 (1-5)
        t1: 反应时 (ms)
        t2: 按键持续时间 (ms)
        ide: 校验值
    """
    with serial.Serial(PORT, BAUD, timeout=TIMEOUT) as ser:
        # 清空缓冲区
        ser.reset_input_buffer()
        
        # 停止之前的计时
        ser.write(b'\xFC\x00')
        
        # 触发计时（在刺激呈现时调用）
        ser.write(b'\xFB\x00')
        
        # 等待按键响应（读取8字节）
        frame = ser.read(8)
        
        # 验证帧格式
        if len(frame) != 8:
            raise RuntimeError(f"超时未按键：{frame.hex(' ')}")
        
        if frame[0] != 0xFB:
            raise RuntimeError(f"帧头错误：{frame.hex(' ')}")
        
        if frame[1] not in {0x01, 0x02, 0x03, 0x04, 0x05}:
            raise RuntimeError(f"按键编号错误：{frame[1]}")
        
        # 解析数据
        res = frame[1]  # 按键编号
        t1, t2 = struct.unpack('>HH', frame[2:6])  # 大端解析时间
        ide = int.from_bytes(frame[6:], byteorder='big')  # 校验值
        
        return res, t1, t2, ide

# 使用示例
key, reaction_time, press_duration, check = get_reaction_time()

print(f"按键: {key}")
print(f"反应时: {reaction_time} ms")
print(f"按压时长: {press_duration} ms")
```

## 6. 测量误差与系统误差

### 6.1 误差来源

#### 系统误差（可预测、可校准）

1. **串口通信延迟**
   - 数据传输时间：约 0.5-2 ms
   - 可通过测试估算并补偿

2. **反应盒硬件延迟**
   - 按键检测延迟：< 1 ms
   - 芯片处理延迟：< 1 ms

3. **软件处理延迟**
   - Python 解析时间：< 1 ms
   - 可忽略不计

#### 测量误差（随机、不可预测）

1. **时间分辨率**
   - 反应盒时钟精度：1 ms
   - 测量误差：±0.5 ms

2. **按键抖动**
   - 机械按键的物理抖动
   - 影响：±1-2 ms

### 6.2 误差测量方法

#### 测试系统延迟
```python
import time

# 测试100次，计算平均系统延迟
delays = []
for i in range(100):
    # 记录发送命令时间
    start = time.perf_counter()
    
    # 发送触发命令
    ser.write(b'\xFB\x00')
    
    # 模拟立即按键（需要硬件配合）
    # 读取响应
    frame = ser.read(8)
    
    # 记录接收时间
    end = time.perf_counter()
    
    # 计算延迟
    delay = (end - start) * 1000  # 转换为毫秒
    delays.append(delay)

# 系统误差估计
system_delay = np.mean(delays)
delay_std = np.std(delays)

print(f"平均系统延迟: {system_delay:.2f} ms")
print(f"延迟标准差: {delay_std:.2f} ms")
```

#### 校准反应时
```python
# 原始反应时
raw_rt = t1  # 毫秒

# 校准后的反应时（减去系统延迟）
calibrated_rt = raw_rt - system_delay

print(f"原始反应时: {raw_rt} ms")
print(f"校准反应时: {calibrated_rt} ms")
```

### 6.3 误差报告

在发表文章时，应报告：

```python
# 计算统计量
rt_mean = np.mean(reaction_times)
rt_std = np.std(reaction_times)
rt_sem = rt_std / np.sqrt(len(reaction_times))

print(f"平均反应时: {rt_mean:.2f} ± {rt_sem:.2f} ms")
print(f"标准差: {rt_std:.2f} ms")
print(f"时间分辨率: 1 ms")
print(f"估计系统误差: ±{system_delay:.2f} ms")
```

## 8. 故障排除

### 8.1 无法找到串口

**问题**：程序提示找不到串口或COM端口

**解决方案**：
1. 检查USB线是否连接
2. 检查驱动是否正确安装
3. 查看设备管理器确认COM口号
4. 尝试重新插拔USB线
5. 更换USB接口

### 8.2 串口无法打开

**问题**：提示 `SerialException: Access denied`

**解决方案**：
1. 确认串口未被其他程序占用
2. 关闭串口调试工具（如putty、Arduino IDE等）
3. 重启Python程序
4. 以管理员权限运行

### 8.3 无数据返回

**问题**：发送命令后无响应

**解决方案**：
1. 检查波特率是否正确（115200）
2. 检查反应盒是否通电
3. 尝试发送查询命令 `b'\x5A\x00'`
4. 增加超时时间
5. 检查数据线是否损坏

### 8.4 数据格式错误

**问题**：接收到的数据帧不符合格式

**解决方案**：
1. 清空串口缓冲区：`ser.reset_input_buffer()`
2. 确认发送的命令正确
3. 检查是否有残留数据
4. 按正确顺序发送命令
